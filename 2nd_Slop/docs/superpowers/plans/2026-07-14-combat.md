# L'Épreuve d'armes — Plan d'implémentation

> Spec : `docs/superpowers/specs/2026-07-14-combat-design.md` (curseurs validés
> par Jael le 2026-07-14). Tâches TDD, `npm test` vert entre chaque, un commit
> par tâche.

**Idée maîtresse : le combat est un sous-deck.** Quand `reign.combat` existe,
`draw()` pioche une **manœuvre** au lieu d'une carte du deck, et `choose()` la
**résout** au lieu d'appliquer des effets de jauges. La boucle de `main.js`
(drag → commit → désintégration → pioche), le clavier et le fuzz traversent
donc le combat sans changement structurel.

## Décisions d'intégration (issues de la lecture du moteur)

- **Déclenchement** : un côté de carte porte `combat: '<id>'`. `choose()` pose
  alors les `flags` du côté (dont `epreuve.<id>` — posé par la CARTE, pas par
  le moteur, pour que l'invariant « tout flag requis est posé par le deck »
  reste vert), ignore ses `effects`, et appelle `startCombat`. **L'année
  n'avance qu'à la fin du combat** (un combat = un an).
- **Signature** : `choose(reign, side, rng = Math.random)` — le rng mélange la
  pioche de manœuvres (fuzz reproductible ; le fuzz passera son mulberry32).
- **Pioche de manœuvres** : `deck = shuffle(specialsÉligibles + communes)` à
  l'entrée ; 5 manœuvres communes sans condition ≥ `maxRounds = 5` → jamais de
  pioche vide. Recyclage des communes en garde-fou.
- **Résolution d'une manche** (déterministe, dans cet ordre) :
  1. `strike` : dégâts + bonus si jauge ≥ seuil ; **+1 si Excalibur portée et
     strike de chevalerie** (la relique tient sa promesse en duel) ;
  2. adversaire mort → victoire (l'audace paie avant la riposte) ;
  3. riposte : `max(0, foe.atk - guard)`, `+expose` si le bonus a été raté
     (contre-coup dans la même manche — la spec est amendée en ce sens) ;
  4. `heal` si jauge ≥ seuil ; 5. défaite si blasons à 0 ; 6. `round++`,
  retraite si `round > maxRounds`.
- **Fin de combat** : défaite d'un combat `fatal` avec le roi → le **Fourreau
  intercepte** (consommé + message miracle, la défaite devient l'outcome
  `lose`) sinon mort dédiée (`dead.cause = def.deathCause`). Tout autre
  résultat applique `outcome.{win,lose,draw}` = effets de jauges (via
  `empowerEffects`) + flags, puis `years++`, ère, `checkDeath` +
  `tryCancelDeath` — même queue que `choose()`, implémentée dans `combat.js`
  (imports directs gauges/flags/relics/config, **pas de cycle avec reign.js**).
- **Champion** : `def.champions` = liste ordonnée `{name, speaker, requires}`,
  premier éligible, repli `{name: 'Le Roi', isKing: true}`. Le portrait passe
  par `portraitFor(speaker)` (inconnu → chevalier anonyme, acceptable v1).
- **Rendu** : la carte de manœuvre suit le chemin standard (drag, labels,
  désintégration). En plus : voile rouge, **carte de l'adversaire** en haut
  (drawCard à échelle réduite via ctx.scale, texte vide, bandeau = nom),
  médaillon du champion en bas à gauche, **blasons** (écus pleins/vides) en
  HUD fixe — adversaire en haut à droite, champion en bas à gauche.
- **Audio** : piste `bataille` + SFX `choc`/`ovation` générés par scripts,
  clés branchées en best-effort (silence si absentes). **WAV non committés** :
  ils rejoignent le lot « verdict d'écoute Jael » avec la passe graves.

## Tasks

### Task 1 — Tests moteur sur un combat de fixture
`test/combat.test.js` : fixture COMBAT_TEST (2 blasons chacun, 1 spéciale
gated par flag, commune de repli). Couvre : terminaison ≤ maxRounds quel que
soit l'enchaînement (mini-fuzz 200 seeds), déterminisme (même seed → même
issue), bonus de jauge, expose/guard/heal, victoire/défaite/retraite +
outcomes appliqués, fatal roi → mort dédiée, fatal + Fourreau → survie +
fourreau consommé, champion par flags avec repli roi, pioche jamais vide.
Commit : `test(logres): combat — résolution, terminaison, champion, Fourreau`.

### Task 2 — Moteur
`src/game/combat.js` : `startCombat(reign, def, rng)`, `nextManoeuvre(reign)`,
`resolveManoeuvre(reign, side)`, `resolveChampion(def, flags)`, fin de combat
(queue année/ère/mort). `src/game/reign.js` : hooks 3 lignes dans `draw()` et
`choose()` (+ paramètre rng). Task 1 passe au vert.
Commit : `feat(logres): moteur de combat — le duel comme sous-deck`.

### Task 3 — Contenu des 4 combats
`src/game/combats/index.js` : registre COMBATS + MANOEUVRES. 5 communes
(charge, garde haute, feinte, prière, cri de guerre), 2-3 dédiées par combat
(sort de Morgane à Camlann si `morgane.cour`, charge de l'ost, etc.), textes
dans la voix du jeu (le second commente le duel). Cartes déclencheuses :
`chute.camlann.duel` (gauche, `-30` remplacé par le combat),
`common.heraut.defi`, `common.saxons.raid`, `camelot.tournoi.grand` — chaque
côté combat pose `epreuve.<id>` et la carte prend `noneFlags` dessus (un
combat par règne ; le côté « refus » laisse la carte revenir).
Commit : `feat(logres): les 4 épreuves d'armes — Camlann, champ clos, ost, tournoi`.

### Task 4 — Invariants et fuzz intégrés
`test/invariants.test.js` : le fuzz passe `rng` à `choose` et traverse les
combats (le garde-fou 5000 suffit) ; nouveaux tests — ids de manœuvres
uniques, tout `combat:` référencé existe dans COMBATS, flags d'outcome
inclus dans l'univers des flags posés (union deck + combats), équilibre :
chaque combat garde ≥ maxRounds manœuvres inconditionnelles.
Commit : `test(logres): invariants combat + fuzz au travers des duels`.

### Task 5 — Mise en scène
`src/render/combat.js` (voile, carte adverse réduite, médaillon champion,
blasons) branché dans `drawPlay` quand `reign.combat`. `scripts/combat-shot.mjs`
(force un combat par code de triche : `#combat=<id>` géré par main.js en mode
dev — retiré ? Non : paramètre volontairement non documenté, inoffensif) pour
les captures. Vérif visuelle : duel lisible, drag et aperçu or/rouge OK.
Commit : `feat(logres): mise en scène des duels — vitraux qui se toisent`.

### Task 6 — Audio (génération, hors commit)
`scripts/music.mjs` : piste `bataille` (percussions martiales, mode sombre) ;
`scripts/sfx.mjs` : `choc`, `ovation`. `main.js` : `m_bataille` pendant le
combat, `choc` sur manœuvre, `ovation` sur victoire (clés best-effort).
Commit du code SEULEMENT (`feat(logres): audio de bataille — clés branchées`),
WAV laissés au lot « verdict d'écoute ».

`npm test` complet, `node scripts/combat-shot.mjs`, partie manuelle poussée
