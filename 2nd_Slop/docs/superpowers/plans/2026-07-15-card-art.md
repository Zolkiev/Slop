# Logres — Cadres par domaine + scènes des moments forts — Implementation Plan

> **For agentic workers:** les tâches de CODE (A3, B2) sont TDD et peuvent être déléguées à un subagent. Les tâches de GÉNÉRATION (A1, A2, B1) sont pilotées par le contrôleur avec **gate visuel de Jael** (directeur artistique) — pas de subagent pour juger l'art. Steps en `- [ ]`.

**Goal:** Donner à chaque carte de Logres un cadre de verrière tinté par le domaine de son orateur (chantier A), puis ajouter ~12 scènes dédiées aux cartes-climax (chantier B).

**Architecture:** A = 5 cadres `assets/ui/plate-<domaine>.png` + une map `speaker→domaine` (`domains.js`) + `plateFor(card)` remplaçant `cardPlate()` aux 3 points de rendu. B = ~12 PNG 128×128 dans `assets/scenes/` + clés dans `SCENES` + `art:` sur les cartes. Le moteur (`drawCard`, `cardArt`) est déjà prêt.

**Tech Stack:** Vanilla JS, Canvas 2D, Vite, Vitest. Génération : `scripts/pixellab.mjs` (PixelLab v2, clé dans `2nd_Slop/.env`, 2333 générations dispo) + recolor local pngjs (0 crédit).

## Global Constraints

- Vanilla JS ESM, zéro dépendance runtime. Canvas logique 480×800 ; carte 340×460 ; cadre source 352×480 ; scènes 128×128.
- **`.env` (clé PixelLab) est gitignoré — ne JAMAIS le committer ni afficher la clé.**
- **Estimation de crédits annoncée à Jael AVANT chaque génération** ; le retenu passe par son **gate visuel**.
- Domaines : `foi | magie | chevalerie | couronne | peuple`. Teintes : Foi=ambre/or, Magie=émeraude/violet, Chevalerie=bleu-acier, Couronne=pourpre/cramoisi, Peuple=terre.
- `src/main.js` et `src/render/*` ne sont PAS unit-testés (convention projet) — vérif = suite verte + build + gate visuel. Les modules purs (`domains.js`) SONT testés.
- Fichiers de sortie PixelLab d'abord dans `assets/gen/…` (candidats tracés), retenu copié vers `assets/ui|scenes/`.
